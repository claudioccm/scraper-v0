# Story 1.4: Content Index Page Generation

**Status:** Approved

## Story

**As a** site visitor,  
**I want** index pages that list all content of each type,  
**so that** I can browse blog posts, case studies, and client work in generated static pages.

## Acceptance Criteria

1. Create `ccmBlogIndex`, `ccmCaseStudyIndex`, `ccmClientIndex` page components
2. Components use `useContentPipeline` to generate static HTML lists during build
3. Each index page includes content metadata and links to detail pages
4. All index pages generate as static HTML files during `nuxt generate`

## Tasks / Subtasks

- [ ] Create `ccmBlogIndex` component (AC: 1, 2)
  - [ ] Implement blog post listing with metadata display
  - [ ] Use `useContentPipeline` composable for data fetching during build
  - [ ] Render semantic HTML list structure with proper heading hierarchy
  - [ ] Include blog post titles, dates, categories, and excerpt snippets
- [ ] Create `ccmCaseStudyIndex` component (AC: 1, 2) 
  - [ ] Implement case study listing with client and service information
  - [ ] Use `useContentPipeline` composable for case study data during build
  - [ ] Render grid layout compatible with existing ccmMasterGrid system
  - [ ] Include case study titles, client names, sectors, and brief descriptions
- [ ] Create `ccmClientIndex` component (AC: 1, 2)
  - [ ] Implement client portfolio listing with project associations
  - [ ] Use `useContentPipeline` composable for client relationship data
  - [ ] Group case studies by client organization for portfolio display
  - [ ] Include client names, sectors, and associated project previews
- [ ] Implement static page generation (AC: 4)
  - [ ] Create pages for blog index (/blog), case study index (/work), client index (/clients)
  - [ ] Ensure all index pages generate correctly during `nuxt generate` process
  - [ ] Validate all generated index pages work correctly in static deployment
  - [ ] Test content links navigation between index and detail pages

### Component API Integration 
**Source**: `types/content.ts`, `composables/useContentPipeline.ts`

**ccmBlogIndex Props**:

```typescript
  categories?: string[]                 // Optional category filtering

  variant?: 'grid' | 'list' | 'featured'
  class?: string
}
```

**ccmCaseStudyIndex Props**:
```typescript  
interface CaseStudyIndexProps {

  variant?: 'portfolio' | 'grid' | 'showcase'
  showImages?: boolean
  class?: string
}
```
